Understanding macro variables in SAS

Why do we need macro variables

  • Macro variables allow values to be defined once and reused throughout a program
  • This makes code more dynamic and easier to maintain
  • If a date, period, dataset name, or label changes, we can update one macro variable instead of editing many lines of code
  • Macro variables are one of the main tools SAS provides for code automation
Concept Definition
Macro variable A macro variable stores text that can be substituted into SAS code dynamically.
User defined macro variable A macro variable created by the programmer to store a custom value.
System defined macro variable A macro variable automatically provided by SAS to hold environment information such as date, time, or user id.

Examples of system defined macro variables

  • SAS automatically provides many useful macro variables
  • Examples include `&SYSDATE`, `&SYSTIME`, and `&SYSUSERID`
  • These values are often used in titles, logs, file names, and audit-style outputs

Use macro variables in titles and labels

  • Macro variables are not limited to data filtering
  • They can also be used in titles, labels, file names, and many other places where SAS expects text

Key points to remember

  • Macro variables store text, not SAS data values in a dataset
  • User defined macro variables are usually created with `%let`
  • System macro variables are provided automatically by SAS
  • Macro variables help with dynamic code, reusable code, and automation
