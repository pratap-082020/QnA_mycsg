First dot concept

Lesson Description
-
  • Many analyses ask for the first record of each subject, first visit of each subject, first event of each term — the idea of picking the first row inside a group.
  • In SAS, once the data is sorted by the grouping variable, the automatic first. flag becomes TRUE on the first row of each group.
  • Tidyverse reaches the same idea with group_by() + slice(1) or row_number().
  • Base R uses !duplicated(group) after sorting to keep the first occurrence of each group.
  • This lesson shows two small examples on CLASS: a within-group counter, and the lowest-Height subject within each Sex.
  • Both examples rely on sorting first — the definition of "first" only makes sense after we pick an order.

*------------------------------------------------------------------------------;
*counter;

proc sort data=class;
     by sex height weight name;
 run;

data counter;
     set class;
     by sex height weight name;
     if first.sex then counter=1;
    else counter+1;
run;

*------------------------------------------------------------------------------;
*subset of lowest height;
*------------------------------------------------------------------------------;

proc sort data=class;
     by sex height;
 run;

data lowestheight;
     set class;
     by sex height;
     if first.sex;
     keep sex height name;
 run;
  • In SAS, once a dataset is sorted by a variable, the DATA step exposes automatic flags called first.var and last.var when that variable is named in a by statement.
  • first.sex is TRUE on the first record of each Sex group and FALSE otherwise — exactly the signal we want for a within-group counter.
  • For the counter example, proc sort data=class; by sex height weight name; sets a stable order, then the DATA step reads CLASS with that same by list.
  • if first.sex then counter=1; else counter+1; uses the sum statement form — counter+1 automatically retains the counter across rows, so we never have to write an explicit retain.
  • For the lowest-Height example, sorting by sex height and writing if first.sex; on its own line keeps only the first row of each Sex — the one with the smallest Height.
  • After running, COUNTER has an incrementing counter within each Sex; LOWESTHEIGHT has one row per Sex with the smallest Height.

  • In tidyverse, the "first row within each group" idea is expressed with group_by() combined with either row_number() or slice(1).
  • For the counter: arrange(Sex, Height, Weight, Name) %>% group_by(Sex) %>% mutate(counter = row_number()) assigns 1, 2, 3, ... within each Sex group after sorting.
  • row_number() inside a grouped mutate is the direct tidyverse equivalent of the SAS first.sex then counter=1; else counter+1; pattern.
  • For the lowest-Height example: arrange(Sex, Height) %>% group_by(Sex) %>% slice(1) %>% select(Name, Sex, Height) returns one row per Sex after sorting by Height ascending.
  • slice(1) keeps the first row in the current group; combined with the earlier sort, that is the row with the smallest Height.
  • We run and confirm the counter climbs from 1 within each Sex, and the lowest-Height tibble has one row per Sex with the smallest Height value.
R (base)
+ 
class <- data.frame(
  Name = c("Alfred", "Alice", "Barbara", "Carol", "Henry", "James", "Jane", "Janet", "Jeffrey", "John", "Joyce", "Judy", "Louise", "Mary", "Philip", "Robert", "Ronald", "Thomas", "William"),
  Sex = c("M", "F", "F", "F", "M", "M", "F", "F", "M", "M", "F", "F", "F", "F", "M", "M", "M", "M", "M"),
  Age = c(14, 13, 13, 14, 14, 12, 12, 15, 13, 12, 11, 14, 12, 15, 16, 12, 15, 11, 15),
  Height = c(69, 56.5, 65.3, 62.8, 63.5, 57.3, 59.8, 62.5, 62.5, 59, 51.3, 64.3, 56.3, 66.5, 72, 64.8, 67, 57.5, 66.5),
  Weight = c(112.5, 84, 98, 102.5, 102.5, 83, 84.5, 112.5, 84, 99.5, 50.5, 90, 77, 112, 150, 128, 133, 85, 112)
  , stringsAsFactors = FALSE
)

#==============================================================================;
#Counter;
#==============================================================================;

 
counter_tmp <- class
counter_tmp <- counter_tmp[order(counter_tmp$Sex, counter_tmp$Height, counter_tmp$Weight, counter_tmp$Name), ]
counter_tmp$counter <- ave(counter_tmp$Sex, counter_tmp$Sex, FUN = seq_along)
counter <- counter_tmp

 
#==============================================================================;
#Subset lowest height student within each sex;
#==============================================================================;

 
lowestheight_tmp <- class
lowestheight_tmp <- lowestheight_tmp[order(lowestheight_tmp$Sex, lowestheight_tmp$Height), ]
lowestheight_tmp <- lowestheight_tmp[!duplicated(lowestheight_tmp$Sex), ]
lowestheight_tmp <- lowestheight_tmp[, c("Name", "Sex", "Height")]
lowestheight <- lowestheight_tmp
  • In base R, the within-group counter is built with ave(): ave(df$Sex, df$Sex, FUN = seq_along) returns a vector that restarts at 1 within each Sex group.
  • ave(x, group, FUN = f) runs f separately on each slice of x defined by group, then lines the results back up row-for-row — the direct base-R analogue of a grouped mutate.
  • Before computing the counter we sort the data frame with order(Sex, Height, Weight, Name) so the counter climbs in a meaningful order.
  • For the lowest-Height example, we sort by Sex and Height ascending, then apply !duplicated(Sex) to keep only the first occurrence of each Sex — which is the lowest-Height row after the sort.
  • df[, c("Name", "Sex", "Height")] then trims the result to the columns the report needs.
  • We run and compare the counter and the one-row-per-Sex data frame against the tidyverse output — they should match row for row.
