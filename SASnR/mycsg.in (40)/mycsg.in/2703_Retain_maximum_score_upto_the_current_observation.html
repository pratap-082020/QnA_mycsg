Reading Files Using Path References

Lesson Description
-
  • Sometimes, we want to work with the concept of "Reading Files Using Path References" in a clear, repeatable way.
  • This lesson walks through a simple example and shows the key steps.
  • We will see one approach on how we can do it in SAS and R.
SAS (Base SAS)
+

 


libname mydata "path"; 

data class; 
    set mydata.class; 
run;

 

  • In SAS, `libname` assigns a shortcut (library) to a folder of datasets.
  • You can then refer to datasets in that folder using the library name, e.g., `mydata.class`.
  • This enables reading multiple datasets from a folder without repeating the full path.
R (base)
+ 
# Direct path reading
library(haven)

class <- read_sas("path/class.sas7bdat")

# Using file.path for portability
folder_path <- "D:/SAS/Home/dev/"
class <- read_sas(file.path(folder_path, "class.sas7bdat"))

# Using glue for portability
library(glue)
folder_path <- "D:/SAS/Home/dev/"
class <- read_sas(glue("{folder_path}/class.sas7bdat"))
  • To read a .sas7bdat file into R, we use haven::read_sas().
  • The simplest form takes a full path: read_sas("path/class.sas7bdat").
  • For portability across machines, we build the path dynamically with file.path() — it inserts the correct separator for the operating system.
  • glue() offers a string-interpolation alternative that many find easier to read: glue("{folder_path}/class.sas7bdat").
  • All three approaches end up in the same spot — a tibble holding the contents of the SAS dataset.
  • This lesson has no separate tidyverse variant because read_sas() from haven is the standard way in any R workflow — tidyverse users and base R users use the same function.
