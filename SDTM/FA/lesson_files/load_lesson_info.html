Background
  • In clinical trials, as per SDTM standard, the data collected is classified into three general observation classes.
    • Interventions
    • Events
    • Findings
  • Sometimes, we may need to collect additional information about the events or interventions
  • As per SDTM standard, we have to represent the additional information collected about events or interventions in a dataset classed as 'Findings About'
  • These 'Findings About' share the same list of variables as that of a SDTM 'Findings' domain but has an additional variable named 'FAOBJ' - objective of the observation or test presented on a record
  • Some examples:
    • Additional details about vomiting in a vaccine study: 
      • Occurrence of vomiting in a vaccine study is an event and we may want to collection additional information about this event in terms of 
        • Number of episodes
        • Volume of vomit
      • We do not have standard variables in events class to represent this information in AE
      • As per SDTM standard, as these are additional questions about an event, we need to use FA domain to present this data
    • Additional details about bone fractures in osteoporosis study
      • Bone fracture is considered an event and any additional information collected about this event has to be presented in FA domain
        • Reason for bone fracture
        • Activity being performed when bone fracture was supposed to have occurred
        • Any further complications occurred - yes/no

What type of information is typically collected or presented?

 

  • Objective for which the test is collected
  • Short name for the test 
  • Name of the test
  • Result
    • in collected units
    • in standard units
  • Date/time of collection
