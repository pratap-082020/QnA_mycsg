Figures in clinical trials are graphical outputs that present data visually to support the interpretation of trial results. They appear in the CSR alongside tables and listings, and are especially important for communicating complex longitudinal, distributional, and survival data to regulatory reviewers.

Common types of figures in a CSR:

  • Kaplan-Meier (KM) survival curves — Time-to-event analysis (e.g., overall survival, progression-free survival). The most frequently required figure in oncology trials.
  • Mean (or LS mean) over time plots — Showing the trajectory of a continuous endpoint (e.g., HbA1c, EASI score, FEV1) over scheduled visits for each treatment arm, with error bars (SE or 95% CI).
  • Box plots — Showing the distribution of a continuous variable (median, IQR, whiskers, outliers) by treatment group and/or visit.
  • Forest plots — Displaying subgroup treatment effects and their confidence intervals on a common scale; used in efficacy subgroup analyses and meta-analyses.
  • Waterfall plots — Showing individual subject response (e.g., percent change from baseline in tumor burden), one bar per subject, commonly used in oncology.
  • Scatter plots — Correlation between two continuous variables (e.g., PK exposure vs. efficacy/safety outcome).
  • Bar charts / Pie charts — Occasional use for categorical distributions, though tables are generally preferred in regulatory submissions.
  • Individual patient profiles — Time course of lab or PK values for individual subjects, common in Phase I.

In SAS, most clinical figures are produced using PROC SGPLOT, PROC SGPANEL, or PROC LIFETEST (for KM curves), rendered via ODS Graphics.

A Kaplan-Meier (KM) curve is used to estimate the survival function from time-to-event data — for example, time to death, time to disease progression, or time to first adverse event. It is the most commonly requested figure in oncology and cardiovascular trials.

Data source: Typically from a time-to-event ADaM dataset such as ADTTE. Key variables: PARAM (which event, e.g., Overall Survival), AVAL (time to event or censoring in days or months), CNSR (censoring indicator: 0 = event, 1 = censored), TRT01A (treatment arm).

Step 1 — Run PROC LIFETEST to estimate the KM curve:

PROC LIFETEST DATA=adtte PLOTS=SURVIVAL(ATRISK=0 TO 24 BY 6 CL)
              OUTSURV=km_est TIMELIST=(6 12 18 24) STDERR;
    TIME aval * cnsr(1);       /* aval = time, cnsr=1 means censored */
    STRATA trt01a;             /* separate curve per treatment arm */
    ODS OUTPUT SURVIVALPLOT=km_plot_data;
RUN;

Step 2 — Customize the figure using PROC SGPLOT or the ODS Graphics output from PROC LIFETEST:

  • Add a number-at-risk table below the x-axis using XAXISTABLE statement in PROC SGPLOT
  • Control line colors, styles, and legend labels via STYLEATTRS or KEYLEGEND statements
  • Add median survival times and 95% CIs as text annotations using INSET or REFLINE
  • Control axis ranges, tick marks, and labels

Key considerations:

  • Confirm the censoring convention — in ADaM ADTTE, CNSR=1 typically means censored (no event); confirm this with the ADaM spec before running
  • The number-at-risk table below the x-axis is a regulatory expectation — always include it
  • Log-rank test p-value is often added to the figure or the accompanying table
  • Output format: typically PNG or EMF embedded in RTF/PDF; confirm sponsor requirements for figure resolution (300 DPI minimum for submission)

A waterfall plot is a bar chart where each bar represents one subject, showing the maximum percent change from baseline in a tumor measurement (typically sum of diameters per RECIST criteria). Bars point upward for subjects with tumor growth and downward for subjects with tumor shrinkage. It is a standard exploratory figure in oncology trials to visualize individual subject responses.

Typical features of a waterfall plot:

  • Each bar = one subject; bars are sorted from greatest decrease to greatest increase (left to right)
  • Horizontal reference lines at −30% (partial response threshold per RECIST 1.1) and +20% (progressive disease threshold)
  • Bars are colored or patterned by response category (CR, PR, SD, PD) or treatment arm
  • Subjects who did not have a post-baseline assessment may be excluded or shown separately — confirm from spec

Programming in SAS with PROC SGPLOT:

/* Prepare data: one row per subject with max pct change */
DATA wf_data;
    SET adrs;
    WHERE paramcd='BESRSPI' AND fasfl='Y';  /* best overall response % change */
    /* Sort order: ascending by pct change */
RUN;

PROC SORT DATA=wf_data; BY pchg; RUN;

DATA wf_data;
    SET wf_data;
    subject_order + 1;
RUN;

PROC SGPLOT DATA=wf_data;
    STYLEATTRS DATACOLORS=(green red blue);
    VBAR subject_order / RESPONSE=pchg GROUP=avalc
         GROUPDISPLAY=CLUSTER NOOUTLINE;
    REFLINE -30 / AXIS=Y LINEATTRS=(COLOR=black PATTERN=DASH);
    REFLINE  20 / AXIS=Y LINEATTRS=(COLOR=black PATTERN=DASH);
    XAXIS DISPLAY=NONE LABEL='Subject';
    YAXIS LABEL='Best % Change from Baseline in Tumor Burden'
          VALUES=(-100 TO 100 BY 20);
    KEYLEGEND / TITLE='Best Overall Response';
RUN;

Key considerations:

  • Confirm whether the best percent change is pre-derived in ADRS or must be derived as the minimum (most negative) post-baseline percent change per subject
  • Subjects with complete responses (100% decrease, tumor not measurable) may need special handling at the −100% floor
  • The sort order of the bars (ascending by pct change) is typically fixed — do not sort alphabetically by subject ID

A forest plot displays the treatment effect (typically an odds ratio, hazard ratio, or mean difference) and its 95% confidence interval for multiple subgroups on a common scale. The overall treatment effect is usually shown at the bottom. It is used in subgroup efficacy analyses to explore consistency of treatment benefit across patient subgroups.

Key components of a forest plot:

  • Left panel: subgroup names and sample sizes (n per arm)
  • Middle panel: the plot — a square (or diamond) for the point estimate and a horizontal line for the 95% CI; a vertical reference line at the null (HR=1.0 or OR=1.0 or difference=0)
  • Right panel: numeric values — point estimate and 95% CI in text form

Programming in SAS — PROC SGPLOT approach:

/* forest_data has: subgroup_label, estimate (HR), lower (CI lower), upper (CI upper), y_order */

PROC SGPLOT DATA=forest_data NOAUTOLEGEND;
    SCATTER Y=y_order X=estimate /
        XERRORLOWER=lower XERRORUPPER=upper
        MARKERATTRS=(SYMBOL=SQUAREFILLED SIZE=10);
    REFLINE 1.0 / AXIS=X LINEATTRS=(PATTERN=SOLID COLOR=GRAY);
    XAXIS TYPE=LOG LABEL='Hazard Ratio (95% CI)'  /* log scale for HR */
          VALUES=(0.1 0.2 0.5 1.0 2.0 5.0);
    YAXIS DISPLAY=NONE;
RUN;

Practical approach — annotated template: Many clinical programmers use a DATA step annotation approach combined with PROC SGPLOT or GTL (Graph Template Language) to produce the full three-panel forest plot, since PROC SGPLOT alone does not easily render the text panels alongside the plot panel. A macro looping over subgroups and using the SAS Annotate facility or GTL LAYOUT LATTICE is a common pattern.

Key considerations:

  • Hazard ratios are plotted on a log scale — use TYPE=LOG on the XAXIS
  • Confidence intervals that extend beyond the plot range (very wide CIs for small subgroups) should be truncated with arrows, not cut off without indication
  • Confirm the direction of the HR (drug vs. placebo, or placebo vs. drug) from the SAP — the interpretation of "favors treatment" direction must be correct and labeled
  • A p-value for treatment-by-subgroup interaction is often added as a column in the right panel

A mean over time plot (sometimes called a mean profile plot) shows how a continuous efficacy or safety endpoint changes over scheduled visits, with separate lines for each treatment arm. Error bars (SE or 95% CI) are usually added to convey variability. These are standard figures for endpoints like HbA1c, FEV1, pain scores, and vital sign parameters.

Data preparation:

  1. Compute mean and SE (or 95% CI bounds) per visit per treatment arm from the analysis dataset (e.g., ADEFF or ADVS)
  2. Usually use observed means for descriptive figures; for model-based figures, use LS means from PROC MIXED output
  3. Include a baseline visit point (AVISIT = 'Baseline', AVISITN = 0 typically)

PROC SGPLOT code structure:

PROC SGPLOT DATA=means_by_visit;
    SERIES X=avisitn Y=mean / GROUP=trt01a
        LINEATTRS=(THICKNESS=2)
        MARKERS MARKERATTRS=(SIZE=8);
    HIGHLOW X=avisitn LOW=lower HIGH=upper / GROUP=trt01a
        TYPE=LINE LINEATTRS=(PATTERN=DASH);
    XAXIS LABEL='Visit (Week)' VALUES=(0 4 8 12 16 24)
          VALUESDISPLAY=('BL' 'Wk4' 'Wk8' 'Wk12' 'Wk16' 'Wk24');
    YAXIS LABEL='Mean Change from Baseline in HbA1c (%)' GRID;
    REFLINE 0 / AXIS=Y LINEATTRS=(COLOR=GRAY PATTERN=DASH);
    KEYLEGEND / TITLE='Treatment';
RUN;

Key considerations:

  • Use AVISITN (numeric visit number) on the x-axis, not AVISIT (character), to ensure correct spacing between visits
  • Specify VALUESDISPLAY= on XAXIS to show the human-readable visit labels even when plotting by numeric visit
  • The error bars should be labeled in the footnote (e.g., "Error bars represent ±1 SE")
  • Number of subjects contributing to each visit point (n per arm per visit) should appear below the x-axis or in a number-at-risk table — confirm from the shell
  • Confirm whether the figure uses observed means or model-derived LS means — they have different interpretations and should be labeled accordingly

A box plot (box-and-whisker plot) displays the distribution of a continuous variable compactly: the median as a center line, the interquartile range (IQR = Q3 − Q1) as the box, the whiskers extending to a defined range (often 1.5×IQR beyond the box), and outlier points beyond the whiskers. In clinical trials, box plots are used for lab values, vital signs, and other continuous safety/efficacy parameters, often displayed by treatment arm and by visit.

PROC SGPLOT code for a box plot:

PROC SGPLOT DATA=advs_boxplot;
    VBOX aval / CATEGORY=avisitn GROUP=trt01a
         GROUPDISPLAY=CLUSTER
         BOXWIDTH=0.6
         WHISKERPERCENTILE=5   /* 5th–95th percentile whiskers */
         OUTLIERATTRS=(SYMBOL=CIRCLEFILLED SIZE=5 COLOR=GRAY);
    XAXIS LABEL='Visit (Week)'
          VALUESDISPLAY=('BL' 'Wk4' 'Wk8' 'Wk12');
    YAXIS LABEL='Systolic Blood Pressure (mmHg)' GRID;
    KEYLEGEND / TITLE='Treatment';
RUN;

Key options:

  • CATEGORY= — defines the x-axis grouping (usually visit number)
  • GROUP= — splits boxes within each category by treatment arm
  • GROUPDISPLAY=CLUSTER — places treatment group boxes side by side within each visit
  • WHISKERPERCENTILE= — defaults to 1.5×IQR; can be set to 5th–95th percentile if specified in the shell
  • Overlaid mean symbol: Add a SCATTER statement on top of the VBOX to show the mean as a distinct symbol (e.g., a diamond) alongside the median box

Common considerations:

  • Add the number of observations per group below the x-axis using XAXISTABLE to give context to the distribution shape
  • Confirm whether outliers should be shown (individual points beyond whiskers) or suppressed — regulatory figures often show them
  • Baseline box should always be included to show the starting distribution before treatment

ODS Graphics is the SAS graphics engine that produces graphical output through the Output Delivery System. It replaced the older SAS/GRAPH procedures (GPLOT, GCHART) as the primary graphics system in modern SAS (SAS 9.2+) and is the foundation for PROC SGPLOT, PROC SGPANEL, PROC SGSCATTER, and the automatic graphs produced by many analytical procedures.

How ODS Graphics is enabled:

ODS GRAPHICS ON / WIDTH=6IN HEIGHT=4IN IMAGEFMT=PNG IMAGENAME='figure_km';
/* ... run PROC LIFETEST, PROC SGPLOT, etc. ... */
ODS GRAPHICS OFF;

Key ODS Graphics settings relevant to clinical figures:

  • WIDTH= and HEIGHT= — control figure dimensions; typical submission requirement is 6×4 inches or 7×5 inches — confirm from sponsor standards
  • IMAGEFMT= — output image format: PNG (screen/RTF), EMF (Windows Enhanced Metafile, scalable and preferred for RTF submissions), or SVG
  • IMAGENAME= — base filename for the image file produced
  • ANTIALIAS=ON — smooth line rendering; ON by default for most destinations
  • BORDER=OFF — removes the outer border around the figure (often preferred for clinical figures)

ODS destination interaction: When producing figures for RTF submission output, ODS Graphics renders the figure and embeds it in the RTF file. For PDF submissions, the figure is embedded in the PDF. The path and filename of standalone image files can be controlled with the GPATH= ODS option.

Figure resolution: Regulatory submissions typically require 300 DPI minimum. Use DPIMAX=300 (or higher) in the ODS GRAPHICS statement to ensure adequate print quality.

Clinical figures submitted as part of a CSR must meet specific formatting standards to ensure readability, regulatory acceptability, and consistency across the submission package. Each sponsor has their own style guide, but common industry standards exist.

Typical formatting requirements:

  • Page orientation: Landscape is standard for most figures to maximize plot area; portrait is acceptable for narrow figures. Confirm per shell.
  • Figure size: 6×4 inches or 7×5 inches (width×height) are common standards for embedded figures in RTF/PDF
  • Resolution: Minimum 300 DPI for print-quality figures
  • Font: Match the sponsor style — typically Arial or Times New Roman, 10–12pt for axis labels and legend, 8–10pt for tick labels
  • Color vs. grayscale: Historically, submissions used grayscale-safe designs (different line patterns/symbols, not just colors, to distinguish groups) since printed copies might be grayscale. Modern PDF submissions accept color. Always ensure line patterns or symbols also distinguish treatment groups for color-blind accessibility.
  • Axis labels: Both axes must be labeled with the parameter name and unit (e.g., "Systolic Blood Pressure (mmHg)"). Axes without units are a QC finding.
  • Titles and footnotes: Must match the approved shell exactly; set using TITLE/FOOTNOTE statements before the ODS/SAS Graphics call
  • Legend: Treatment group labels must match the labels used in the companion tables; placed inside the plot area or below the x-axis — confirm from shell
  • Number-at-risk / n per group: For time-to-event and longitudinal figures, the number of subjects contributing at each time point must be shown

Verification: Always render the final figure at full size in the RTF or PDF viewer and check that no text is cut off, axis tick labels are not overlapping, and the legend is fully visible before locking the output.

A scatter plot is used in clinical trials to visualize the relationship between two continuous variables. A common application is a PK/PD (pharmacokinetic/pharmacodynamic) exposure-response plot showing the relationship between drug exposure (e.g., AUC or Cmax) and an efficacy or safety outcome (e.g., change in HbA1c, incidence of a side effect).

Data preparation: One row per subject with the exposure metric (from ADPC or ADEX) and the outcome measure (from the efficacy or safety dataset), typically at the end-of-treatment or primary endpoint visit, merged by USUBJID.

PROC SGPLOT code:

PROC SGPLOT DATA=pk_pd_data;
    SCATTER X=auc_last Y=chg_hba1c / GROUP=trt01a
            MARKERATTRS=(SIZE=8 SYMBOL=CIRCLEFILLED);
    REG     X=auc_last Y=chg_hba1c / GROUP=trt01a
            LINEATTRS=(THICKNESS=2 PATTERN=SOLID)
            NOMARKERS CLM;   /* regression line with CI band */
    REFLINE 0 / AXIS=Y LINEATTRS=(COLOR=GRAY PATTERN=DASH);
    XAXIS LABEL='AUClast (ng·h/mL)' GRID;
    YAXIS LABEL='Change from Baseline in HbA1c (%)' GRID;
    KEYLEGEND / TITLE='Treatment';
RUN;

Key PROC SGPLOT statements for scatter plots:

  • SCATTER — plots individual points; use GROUP= to color/shape by treatment arm
  • REG — overlays a regression line and optionally a confidence interval band (CLM)
  • LOESS — overlays a non-parametric smooth trend line; useful when the relationship is non-linear
  • PBSPLINE — penalized B-spline smoother, another option for non-linear trend overlay

Considerations:

  • Label axis with variable name and unit — "AUClast (ng·h/mL)" not just "AUClast"
  • Add Pearson or Spearman correlation coefficient as an annotation (INSET statement) if requested in the spec
  • Extremely large or small axis values may require log transformation — use TYPE=LOG on the axis if specified

PROC SGPANEL produces a panel of graphs — multiple plots arranged in a grid, sharing the same axis scales — from a single procedure call. It is used when you want to produce the same plot type for multiple parameters or subgroups simultaneously, with each panel showing one parameter or subgroup.

When PROC SGPANEL is preferred over PROC SGPLOT:

  • Multiple lab parameters on one page: Instead of running a separate PROC SGPLOT for each lab parameter (e.g., ALT, AST, Bilirubin, ALP), PROC SGPANEL produces all panels in a single figure with shared axes, making trends across parameters easy to compare visually
  • Subgroup faceting: When the same plot needs to be shown for multiple subgroups (e.g., male vs. female, age groups) in a compact layout
  • Consistency: All panels automatically use the same axis scale (or optionally independent scales), making cross-panel comparison valid

Basic PROC SGPANEL syntax:

PROC SGPANEL DATA=advs_panel;
    PANELBY paramcd / ROWS=2 COLUMNS=2 NOVARNAME ONEPANEL;
    SERIES  X=avisitn Y=mean / GROUP=trt01a
            LINEATTRS=(THICKNESS=2) MARKERS;
    ROWAXIS LABEL='Mean Value';
    COLAXIS LABEL='Visit (Week)' VALUES=(0 4 8 12 24);
RUN;

Key PANELBY options:

  • ROWS= and COLUMNS= — control the grid layout
  • ONEPANEL — forces all panels onto a single page regardless of count
  • UNISCALE=ROW/COLUMN/ALL — controls whether panels share axes (ALL = same scale on all panels for direct comparability)
  • NOVARNAME — suppresses the default "paramcd = ..." label in panel headers; use HEADERATTRS or a format to show clean parameter names

A common clinical use is a 2×2 or 3×3 panel figure showing mean over time for each vital sign or hematology parameter — one figure, all parameters visible, ready for the CSR appendix.
